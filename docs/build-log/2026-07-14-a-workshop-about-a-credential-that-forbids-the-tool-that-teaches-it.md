The idea sounded simple going in: Anthropic launched real certifications for Claude this year, so build the workshop equivalent of Borrow Native for that. Learn the material the agent-native way, the way this whole factory already teaches everything. What that idea skipped past is that the material here isn't a language with a compiler that doesn't care who wrote the code. It's an exam that explicitly forbids the tool the workshop's entire teaching method is built on. Closed-book, no AI assistance, workspace clear of everything but you. Building toward that with an agent, using an agent, is a real tension, not a marketing hook I bolted on afterward.

The first surprise wasn't the tension. It was that the target I'd scoped, "Claude Developer, Foundations then Professional," doesn't exist. Four live exams as of today: Associate-Foundations, Developer-Foundations, Architect-Foundations, and Architect-Professional. Developer stops at Foundations. Only Architect has both levels. That's the second time this factory has hit a certification that isn't what it first looked like from a marketing page (Borrow Native's Rust Foundation correction was the first), and it's starting to look less like bad luck and more like a pattern worth trusting: check the actual credentialing body's own primary documentation before designing a curriculum around what a landing page implies.

Which led to the second surprise. The first research pass, working from secondary "exam prep" sites, returned a five-domain blueprint that looked identical whether the page claimed to be about Developer-Foundations or Architect-Foundations. Same domains, same percentages, different exam names. Either something was copied, something was scraped wrong, or two different things really do share a blueprint, and there was no way to tell from those sources alone. Fetching Anthropic's own exam guide PDFs directly settled it in one read: real, detailed, thirty task statements for Foundations alone, distinct and much more specific than anything the secondary sites had. That's now this workshop's actual curriculum anchor, not a first pass I'll clean up later.

The review panel's sharpest catch was mundane and would have shipped anyway if nobody looked: the README's tagline still said "Claude Developer" after the correction had already been written into the decision log two files over. Two different personas caught it independently, which is exactly the kind of signal this panel is supposed to produce and exactly the kind of thing a single reviewer, especially the one who wrote the correction, tends to stop seeing once they've internalized it as already fixed. The other finding worth keeping honest about: I'd written "the two-tier gate is the resolution" as a section header, then hedged it as an unproven hypothesis twenty lines later. A reader skimming headers gets the confident version. Fixed, but it's a reminder that where a hedge sits matters as much as whether it exists.

What's actually here today: a scaffolded public repo, a chosen name, one review pass with real findings applied, ten module skeletons (six for Foundations, four for Professional, the split itself a finding, not a design goal, since the two exams have five and seven real domains that don't compress into a clean four-and-four), and no content. Nobody has attempted a closed-book checkpoint against anything in this repo yet, and neither exam has been sat. That's next, and it isn't this session's job to rush.
