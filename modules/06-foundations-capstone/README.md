# Module 06: Foundations Capstone — Sit-Ready for CCA-F

## The question this module answers

Are you actually ready to sit the real Claude Certified Architect – Foundations exam?

## Where it sits in the arc

Sixth module, closing Part 1 (Architect Foundations). Hard prerequisite: all of [Module 01](../01-configuring-claude-code/README.md)–[Module 05](../05-context-reliability/README.md) — this module synthesizes all five CCA-F domains rather than testing any one in isolation. A learner could stop here, sit the real CCA-F exam, and treat Part 2 as a separate, later commitment. Next, if continuing: [Module 07, Designing the Solution](../07-solution-design-context-strategy/README.md), which opens Part 2 (Architect Professional). See [`modules/README.md`](../README.md).

## Learning objectives (placeholder — finalized when content is authored)

- Diagnose, under real exam-scenario framing, which of the five CCA-F domains a given production problem actually belongs to.
- Complete a full closed-book mock exam at real exam pacing (120 minutes) and correctly self-score against the 720/1000 threshold.
- Defend a root-cause diagnosis in writing against a real seeded multi-domain scenario, not just fix it.

## Exercise material to draw from (not a spec — Coachgremlin authors the real exercise later)

The CCA-F Exam Guide itself, in full: its 6 published scenarios (Customer Support Resolution Agent, Code Generation with Claude Code, Multi-Agent Research System, Developer Productivity with Claude, Claude Code for CI, Structured Data Extraction) and its 12 sample questions, used as a style/difficulty reference only — never copied or lightly reworded into this module's own mock exam (see [`docs/design-tension.md`](../../docs/design-tension.md)'s Constraint section). See [`docs/workshop-design.md`](../../docs/workshop-design.md).

## Required gate (placeholder — shape decided now, real rubric written later)

- **Deterministic tier (hands-on, with Claude Code):** a real seeded scenario spanning 3+ CCA-F domains, diagnosed and fixed hands-on, with the fix verified to hold.
- **Exam-condition tier (closed-book, without Claude Code):** a full mock exam mirroring the real exam's own structure — 4 scenarios drawn at random from a pool of 6 (matching CCA-F's real 6-scenario/4-drawn format), roughly 60 questions, 120 minutes, 720/1000 to pass. This is the one module where the checkpoint format matches the real exam exactly rather than the workshop's smaller per-module default, since its entire purpose is a faithful dress rehearsal.

## Takeaway

A personal CCA-F exam-day prep sheet, built from the mock exam's own weak spots — not a generic study guide, but a record of what this specific learner's attempt actually revealed. Packaged by Coachgremlin once the rubric is met.

## Stop condition (placeholder)

The learner's seeded-scenario diagnosis is correct and defended in writing, the mock exam is completed at real pacing and scored, and Coachgremlin confirms both before this module counts as done. Passing the mock exam is not the same claim as passing the real one — see [`docs/workshop-design.md`](../../docs/workshop-design.md)'s dogfooding commitment for how this workshop tests that claim for real.
